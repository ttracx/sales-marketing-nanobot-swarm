Overview

The Sales & Marketing Nanobot Swarm is a hierarchical AI agent platform that orchestrates specialized sub-agents to execute complex marketing and sales workflows autonomously. Built on FastAPI with OpenAI-compatible LLM backends (Ollama / NVIDIA NIM / OpenAI), it exposes both a rich REST API and a web dashboard.

Quick Start

Get the swarm running in under 5 minutes.

1

Get an LLM Backend

Install Ollama for local inference (free) or set up an OpenAI API key.

# Option A: Local Ollama (macOS)
brew install ollama
ollama pull llama3.3
ollama serve

# Option B: Use your OpenAI key (set in .env later)
# AI_API_BASE_URL=https://api.openai.com/v1
# AI_API_KEY=sk-...
2

Clone & Install

git clone https://github.com/ttracx/sales-marketing-nanobot-swarm.git
cd sales-marketing-nanobot-swarm
./scripts/setup.sh

# Edit your environment variables
cp .env.example .env
nano .env
3

Run Your First Swarm

# Start the server
./scripts/launch.sh
# â†’ Running on http://localhost:8200

# Run a swarm task
curl -X POST http://localhost:8200/swarm/run \
  -H "Content-Type: application/json" \
  -d '{
    "task": "Write a LinkedIn content calendar for a B2B SaaS company",
    "team": "content_creation"
  }'

POST /swarm/run

Execute a multi-agent swarm run against a specified team.

POST /swarm/run Dispatch a task to an agent team

Request Body

FieldTypeRequiredDescription
taskstringRequiredNatural language task description
teamstringOptionalTeam name (default: full_funnel)
contextobjectOptionalKey-value context injected into agent prompts
asyncbooleanOptionalReturn run_id immediately for polling (default: false)
max_agentsintegerOptionalOverride concurrent agent limit for this run
thread_idstringOptionalSession ID for persistent conversation context

POST /v1/chat/completions

OpenAI-compatible chat completions endpoint. Drop-in replacement for any OpenAI client by changing the base URL.

POST /v1/chat/completions OpenAI-compatible completions

Accepts the full OpenAI chat completions request schema. The model field is forwarded to your configured LLM backend. Use swarm/<team_name> as the model name to route through the swarm orchestrator.

# Use with any OpenAI client
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8200/v1",
    api_key="your-secret-key"
)

response = client.chat.completions.create(
    model="swarm/full_funnel",   # routes to Full Funnel Swarm
    messages=[{"role": "user", "content": "Plan a product launch campaign"}]
)

GET /v1/models List available models & swarm teams

GET /swarm/teams & GET /swarm/teams/{name}

GET /swarm/teams List all registered teams

Returns metadata for all available agent teams including name, description, agent count, and example prompts.

Append /{team_name} to get full detail including the team's agent roster, tool list, and configuration parameters.

Create a named team from a list of agent names (built-in or custom). The team is immediately available via /swarm/run using the team name you specify.

Deploy

Vercel automatically builds and deploys. Visit your deployment URL + /health to verify.

Render

# render.yaml (included in repo)
services:
  - type: web
    name: sm-nanobot-swarm
    runtime: python
    buildCommand: pip install -e .
    startCommand: uvicorn nanobot.api.gateway:app --host 0.0.0.0 --port $PORT

Click the Render deploy button in the README, connect your GitHub, and set environment variables in the Render dashboard.

Railway

# Procfile (included in repo)
web: uvicorn nanobot.api.gateway:app --host 0.0.0.0 --port $PORT

Railway auto-detects the Procfile. Connect your repo, add env vars, and deploy. Railway provides persistent storage and zero-downtime deploys.

Docker

# Run with Docker Compose (includes Ollama sidecar)
docker compose up -d

# Or build & run manually
docker build -t sm-nanobot-swarm .
docker run -d \
  -p 8200:8200 \
  -e AI_API_KEY=$AI_API_KEY \
  -e AI_API_BASE_URL=$AI_API_BASE_URL \
  --name sm-nanobot-swarm \
  sm-nanobot-swarm

The docker-compose.yml in the repo starts an Ollama container alongside the API server and pre-pulls llama3.3.
