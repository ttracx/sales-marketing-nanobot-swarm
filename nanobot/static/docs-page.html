Section 01

Overview

What the Sales & Marketing Nanobot Swarm is, how it works, and why it exists.

What is the Sales & Marketing Nanobot Swarm?

The Sales & Marketing Nanobot Swarm is an autonomous AI agent platform designed to execute complex sales and marketing tasks without manual intervention. Rather than a single large model generating a monolithic response, the swarm decomposes every incoming goal into specialized sub-tasks and routes each task to one of ten purpose-built agent teams — each staffed with domain-expert AI agents that collaborate in real time using either hierarchical orchestration or flat peer collaboration.

At its core, the system is a goal-first API. You provide a plain-English marketing objective — "Generate 50 qualified leads in the enterprise SaaS segment using BANT criteria" or "Write a 5-email nurture sequence for mid-market prospects" — and the swarm autonomously selects the best team, configures the agents, executes the pipeline, and returns a structured, actionable result. The entire loop runs in 10–25 seconds on standard hardware and scales horizontally on serverless infrastructure.

Built on FastAPI and powered by Ollama Cloud (primary) and NVIDIA NIM (fallback), the Nanobot Swarm is infrastructure-agnostic and deployable in minutes to Vercel, Render, Railway, Docker, or GitHub Codespaces. It exposes both a native REST API and a fully OpenAI-compatible endpoint, making it a drop-in AI backend for any existing marketing tech stack. The platform is open-source under the MIT license, with a Pro tier launching soon for teams requiring CRM integrations, white-label branding, and advanced analytics.

Key Features

  • 10 Specialized Agent Teams — Each team is a pre-built collection of AI agents with domain expertise, purpose-built tools, and optimized prompts for a specific marketing function.
  • Intelligent Auto-Routing — The Team Router analyzes your goal text and automatically selects the most appropriate team using keyword extraction and semantic similarity matching.
  • Dual LLM Backend — Runs Ollama Cloud (ministral-3b:8b) as primary with automatic failover to NVIDIA NIM (llama-3.3-70b-instruct) for high-stakes or complex tasks.
  • OpenAI-Compatible API — Drop-in replacement for OpenAI's chat completions endpoint; works with any OpenAI SDK by changing one URL.
  • Hierarchical and Flat Modes — Hierarchical teams use an orchestrator agent to delegate sub-tasks; Flat teams use peer collaboration with no single point of authority.
  • 7 Marketing Calculation Tools — Built-in utilities for lead scoring, campaign analytics, SEO analysis, email optimization, market segmentation, and ROI calculation.
  • Serverless-First Architecture — Stateless, horizontally scalable, deploys to Vercel Edge in under 2 minutes with zero configuration needed.
  • Dynamic Agent and Team Builder — POST /agent/build and POST /team/build let you create custom agents and teams at runtime via the API.
  • Full Structured Logging — Every swarm run emits structured logs with agent decisions, tool calls, token counts, and latency breakdowns for debugging and auditing.
  • MIT Licensed, Open Source — Fork it, extend it, self-host it. The Pro tier adds CRM integrations, white-labeling, and team scheduling on top of the open-source core.

Docker Compose (Self-Hosted)

Full-control deployment on your own infrastructure. Requires Docker 24+ and Docker Compose v2.

docker-compose.yml
version: "3.9"
services:
  swarm-api:
    image: ghcr.io/ttracx/sales-marketing-nanobot-swarm:latest
    ports:
      - "8000:8000"
    environment:
      OLLAMA_API_KEY: ${OLLAMA_API_KEY}
      NVIDIA_API_KEY: ${NVIDIA_API_KEY}
      GATEWAY_API_KEY: ${GATEWAY_API_KEY}
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 15s
Start commands
git clone https://github.com/ttracx/sales-marketing-nanobot-swarm.git
cd sales-marketing-nanobot-swarm
cp .env.example .env   # then add your OLLAMA_API_KEY
docker-compose up -d
curl http://localhost:8000/health

POST /v1/chat/completions OpenAI-compatible chat completions +

A drop-in replacement for OpenAI's chat completions API. Change only your base_url — all other OpenAI SDK code works as-is. The swarm auto-detects the best agent team from the user message content and routes accordingly.

Python — openai SDK
from openai import OpenAI

client = OpenAI(
    base_url="https://sales-marketing-nanobot-swarm.vibecaas.app/v1",
    api_key="your-gateway-api-key"  # or "none" for the public demo
)

response = client.chat.completions.create(
    model="ministral-3b:8b",
    messages=[{
        "role": "user",
        "content": "Write a 3-email cold outreach sequence targeting VP of Sales at Series B SaaS companies"
    }]
)
print(response.choices[0].message.content)
curl
curl -X POST \
  https://sales-marketing-nanobot-swarm.vibecaas.app/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "ministral-3b:8b",
    "messages": [{"role":"user","content":"Generate a competitive battlecard vs HubSpot for a mid-market CRM sale"}]
  }'

Install Dependencies
pip install -r requirements.txt
3
Set Codespaces Secrets

Complete reference for all environment variables. Start by copying .env.example from the repository root as your template, then fill in your values.

Core LLM & API Keys

VariableRequiredDefaultDescription
OLLAMA_API_KEYRequired—Ollama Cloud API key. Primary LLM backend for all swarm runs. Obtain from cloud.ollama.ai.
OLLAMA_BASE_URLOptionalhttps://api.ollama.aiOverride the Ollama endpoint. Use for self-hosted Ollama or any OpenAI-compatible API.
NVIDIA_API_KEYOptional—NVIDIA NIM API key. Enables automatic fallback to llama-3.3-70b-instruct for high-accuracy tasks.
GATEWAY_API_KEYOptional—When set, all API requests must include Authorization: Bearer YOUR_KEY. Secures your production endpoint.
ANTHROPIC_API_KEYOptional—Enables Claude as an additional LLM backend option. Pro feature.
DEFAULT_MODELOptionalministral-3b:8bDefault model used when none is specified in the API request body.
MAX_LOOPSOptional3Global maximum agent reasoning loops. Caps per-request num_loops to prevent runaway costs.
